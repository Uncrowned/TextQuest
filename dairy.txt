12.11.13
В файле example.xml будет пример формата xml для игр.

chapters - главы, каждая глава может содержать в себе несколько scene.
scene - сцена, единица отображения в виджете, каждая сцена может содержать фоновую картинку, сопровождаемый звук, и абзацы текста,
	а так же параметр name, для перехода к этой сцене из тега choice.
p - абзац, просто текст, по клику мыши переходит к следующему тегу.
sound - звуковой фаил проигрываемый до появления следующего тега sound, или пока просто не закончится. 
	Пустой тег просто прерывает текущий звук, если такой проигрывается.
	В режиме пропуска, или если галочка в настройках не стоит, игнорируется.
image - фоновая картинка, действует до отмены таким же тегом, слово default внутри этого тега означает стандартный игровой фон.
choice - строка выбора, параметр name указывате на сцену, которая будет отображаться следующей. 
	Параметр file указывает на фаил в котором содержится дальнейший текст, если этого параметра нет, то поиск происходит по текущему файлу.

Все теги читаются последовательно, т.е если будет так 
<p>Здесь какой то текст</p>
<image>image.png</image>
То картинка установится как фон только после того как абзац <p> будет прочитан и пользователь нажмет левую кнопку мыши.

Возможно надо будет добавить тег <readed/> который будет добавляться программой, после прочтения пользователем этой сцены.
